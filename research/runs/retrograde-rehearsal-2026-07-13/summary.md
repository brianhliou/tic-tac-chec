# Full-address retrograde rehearsal — 2026-07-13

## Provenance

- Source commit: `5c2442997d0bc3f387b2f01debeafc5a6408c632`
- Command pattern: `cargo run --manifest-path solver/Cargo.toml --release --bin retrograde_rehearsal -- <resolved-children> 12`
- Rules: original Dream Green rules with travel-direction pawn captures
- Rust: `rustc 1.92.0 (ded5c06cf 2025-12-08)`
- OS/architecture: macOS 26.5.2, `arm64`
- Host: Apple M3 Max with 48 GiB RAM (operator-provided hardware description)
- Build profile: optimized, LTO enabled

This is a bounded propagation rehearsal, not a partial solve. It allocates the
entire 2,462,360,745-state post-opening value and remaining-child arrays, then
alternates synthetic resolved win/loss child values over deterministic random
real game IDs. Every parent is generated by the production inverse-geometry,
forward-validated predecessor kernel. Updates use actual dense parent IDs and
append newly resolved parents to a `u32` frontier.

The counters begin at twelve to exercise decrement traffic without claiming
that the sampled parents have been initialized with their exact outdegrees.
Consequently, no game value or W/L/D count can be inferred from this run.

The 4.587 GiB table payload produced 4,698.1 MiB peak RSS before propagation.
Even this intentionally large frontier raised peak RSS by only about 113 MiB.

## Interpretation

The stable integrated cost of about 551 ns/predecessor is 27% above the 433
ns standalone predecessor baseline. Full-address random value/counter access
and frontier insertion are material but do not displace edge generation as the
single-thread bottleneck.

The simple byte arrays and `u32` sparse frontier therefore remain appropriate
for the first solver. The next experiment should use atomic byte updates with
thread-local output frontiers at 1, 2, 4, 8, and performance-core-count threads.
If scaling flattens early, compare rank-sharded owner-computes batching before
implementing the production parallel solver.

## Correctness and limits

Unit tests verify the update state machine: loss children resolve a parent win
once, winning children decrement the counter, counter zero resolves a loss,
and resolved parents ignore later updates. The predecessor kernel retains its
separate reference-engine and forward-replay checks.

This run does not initialize exact outdegrees, process a closed frontier,
compute draws, measure remoteness, or audit minimax values. Alternating child
values and uniform random child IDs are controlled stress inputs, not a model
of the real frontier distribution. It measures one thread only and performs no
checkpoint I/O.
